Privacy Policy

Last updated: August 2025

1) Overview

Torn Chain Panel helps authorized Torn players fetch and manage chain numbers with alerts and faction queue control. This policy explains what data the extension processes, when anything leaves your device, and your choices.

2) Data We Process

A. Stored locally in your browser

  • Verified Torn username – used to namespace settings and fetch your chain number.
  • Admin PIN and session unlock state – protects admin features (localStorage/sessionStorage).
  • Authorization token (if you use one) – saved in localStorage for protected backend routes.
  • UI preferences & state – volume, alert thresholds, flashing settings, panel position, and any queue entries you add.

B. Sent over the network (only when you trigger an action)

  • Queue / chain actions to https://tornpanel.online (and subdomains):
    What is sent: your username and the action (e.g., enqueue, leave, status).
    Purpose: atomically manage chain positions and return your current status.
    Retention: operational logs may include your username and request metadata. Target retention is ≤30 days, used only for reliability and abuse prevention.
  • Optional YATA import from https://yata.yt/api/v1/faction/members/:
    What is sent: the API key you provide, in the request you initiate.
    Purpose: to fetch faction members for convenience inside the UI.
    Retention: the key is not sent to our server; it’s used client-side for your YATA request and not stored by us.

The extension does not collect browsing history or unrelated page content. Content scripts run only on the declared domains.

3) Domains We Contact

  • https://www.torn.com – page where the UI renders.
  • https://tornpanel.online and https://*.tornpanel.online – chain backend and UI bridge.
  • https://yata.yt – optional, only when you press “Load Members” with your API key.

4) Chrome Permissions (and why)

  • storage – save your local preferences and state (volume, alerts, PIN, token, queue list).
  • alarms – run timer/alert logic reliably.
  • tabs – send queue updates from the service worker to your open Torn/TornPanel tabs.
  • Host permissions – limit scripts and network access to the domains listed above.

We do not use advertising or analytics permissions, and we do not request activeTab or scripting.

5) Sharing, Sale, and Tracking

  • We do not sell user data.
  • We do not share data with third parties except as required to fulfill your explicit requests (e.g., your call to YATA).
  • We do not use ads or third-party analytics trackers.

6) Security

  • All network requests use HTTPS.
  • Sensitive local data (PIN, tokens) are stored on your device in localStorage/sessionStorage.
  • Admin unlock is session-scoped.
  • Server endpoints accept only specific actions and domains.

7) Your Choices & Data Deletion

  • Clear local data: remove items via browser dev tools (Application → Storage) or uninstall the extension.
  • Revoke server access: remove/rotate your authorization token; stop using “Get My #” and queue actions.
  • YATA key: remove it from the UI/session to stop YATA requests.
  • Server logs: request deletion via the contact below.

8) Data Minimization

We process only what’s necessary to provide chain queue functionality and the optional YATA import. We do not process sensitive categories like health, precise location, or financial data.

9) Children’s Privacy

This extension is intended for general audiences of Torn players and is not directed to children.

10) Changes to This Policy

If we materially change this policy, we’ll update the “Last updated” date and publish the new version at the same URL.
